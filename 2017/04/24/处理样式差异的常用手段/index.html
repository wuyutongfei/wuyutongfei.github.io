处理样式差异的常用的三种方式

第一种 reset.css 或者 normalize.css

★ Normalize 相对「平和」，注重通用的方案，重置掉该重置的样式，保留有用的 user agent 样式，同时进行一些 bug 的修复，这点是 reset 所缺乏的。

★ Reset 相对「暴力」，不管你有没有用，统统重置成一样的效果，且影响的范围很大，讲求跨浏览器的一致性。

★ 共同点： 是一个很小的CSS文件,在默认的HTML元素样式上提供了跨浏览器的高度一致性

· Normalize 给我的感觉就是不讲求样式一致，而讲求通用性和可维护性。这点可以从详细的注释和模块化的结构体现出来。

· 另外还有一点是，据说 reset 会破坏一些 debugging tool 的样式，这我没碰到过。

第二种 渐进增强：尽可能以低版本浏览器的样式为基础进行设计

第三种 优雅降级：尽量保证高版本浏览器的样式显示，必要时使用hack
